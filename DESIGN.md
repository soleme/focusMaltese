# Focus Maltese Design Guide

## Reference
- 기준 이미지: `assets/sample1.png`
- 현재 저장소에는 `sample.png`가 아니라 `sample1.png`가 존재하므로 이 파일을 기준으로 디자인 가이드를 정의한다.

## Design Direction
- 전체 톤은 따뜻하고 부드러운 `힐링 + 동기부여 + 캐릭터 육성` 방향으로 유지한다.
- 화면은 게임 UI보다 `감성적인 생산성 앱`에 가깝게 설계한다.
- 캐릭터는 과하게 장식하지 않고, 상태 변화와 보상 순간만 분명하게 강조한다.
- 기본 인상은 `깨끗한 카드형 인터페이스 + 동화책 같은 말티즈 일러스트`로 맞춘다.

## Visual Keywords
- warm
- soft
- clean
- friendly
- playful
- reward-driven

## Color System
기본 팔레트는 아래 방향으로 유지한다. 정확한 hex 값은 구현 시 시안 기반으로 미세 조정할 수 있지만, 색 역할은 고정한다.

- Background
  - 따뜻한 크림톤 오프화이트
  - 약한 베이지 그라데이션 또는 소프트 글로우 허용
- Surface
  - 흰색 또는 아주 옅은 아이보리 카드
- Text Primary
  - 진한 웜 그레이 또는 브라운 계열
- Text Secondary
  - 중간 명도의 웜 그레이
- Idle Accent
  - 세이지 그린
- Focusing Accent
  - 맑은 하늘색 또는 민트 블루
- Success Accent
  - 골드, 허니 옐로우
- Fail Accent
  - 톤다운된 코랄, 로즈, 더스티 레드
- Neutral Button
  - 스톤 그레이

## Typography
- 전체 인상은 둥글고 읽기 쉬운 산세리프 계열로 유지한다.
- 제목은 짧고 강하게, 본문은 친근하게 읽히는 톤을 사용한다.
- 숫자 타이머는 가장 중요한 정보이므로 가장 두껍고 크게 노출한다.
- 상태 문구는 타이머보다 약하지만 충분히 눈에 띄게 배치한다.

텍스트 우선순위는 아래 순서를 따른다.
- 1순위: 남은 시간
- 2순위: 상태명 또는 보상 결과
- 3순위: 캐릭터 정보, 레벨, 재화 수치
- 4순위: 보조 안내 문구

## Layout Principles
- 첫 버전은 `단일 세로 카드 화면` 구조로 설계한다.
- 콘텐츠는 중앙 정렬을 기본으로 한다.
- 상단 정보, 캐릭터 영역, 타이머, 액션 버튼 순서로 배치한다.
- 요소 수를 늘리기보다 `여백`으로 안정감을 만든다.

권장 레이아웃 순서:
- 상단: 레벨, 이름, Treat 수량
- 중단: 캐릭터 카드 또는 상태 일러스트 영역
- 중하단: 큰 타이머 숫자
- 하단: 상태에 따른 주요 버튼 1개 또는 모드 선택 버튼들

## Card Style
- 메인 카드 컨테이너는 큰 라운드 코너를 사용한다.
- 표면은 매끈한 화이트 카드, 그림자는 부드럽고 넓게 깔린다.
- 테두리는 강하게 쓰지 않고 그림자와 배경 대비로 분리한다.
- 상태별 카드 배경은 전체를 바꾸지 말고, 내부 일러스트 패널이나 강조 영역에만 반영한다.

## Character Treatment
- 말티즈는 화면의 감정 전달 중심이다.
- 일러스트는 과도한 디테일보다는 `둥근 실루엣, 큰 표정, 짧은 동작선`으로 감정을 표현한다.
- 상태마다 자세와 표정이 즉시 구분되어야 한다.

상태별 기준:
- Idle
  - 편안하게 쉬거나 졸고 있는 자세
  - 포근한 중립 배경
- Focusing
  - 또렷한 눈빛, 살짝 긴장된 자세
  - 맑은 블루 계열 배경
- Success
  - 점프, 미소, 반짝임, 축하 요소
  - 골드 계열 배경과 장식 사용 가능
- Fail
  - 축 처진 귀, 시무룩한 표정
  - 붉은 톤 배경이지만 자극적이지 않게 처리

## State-Specific UI Rules
상태마다 화면이 바뀌더라도 정보 구조는 크게 흔들지 않는다.

- Idle
  - 시간 preset 버튼을 여러 개 노출 가능
  - 시작 전 준비감이 느껴지도록 안정적인 배치 유지
- Focusing
  - 버튼 수를 줄이고 방해 요소를 최소화
  - 메인 액션은 `포기` 또는 `중단` 하나만 둔다
  - 집중 상태 문구를 색상으로만 가볍게 강조한다
- Success
  - 가장 강한 시각적 보상 상태
  - XP, Treat 보상 카드 분리 노출
  - 수집 버튼은 금색 계열 CTA로 강조
- Fail
  - 실패 표현은 부드럽고 비난하지 않게 유지
  - 재시도 버튼을 명확히 노출
  - 타이머 숫자는 희미하게 낮춰 여운을 주는 방식이 적합

## Buttons
- 버튼은 둥글고 손에 잡히는 장난감 같은 인상을 준다.
- 기본 버튼은 라운드 직사각형 또는 캡슐형을 사용한다.
- 상태별 주요 CTA는 한 화면에 하나만 강하게 보이게 한다.
- 버튼 텍스트는 짧고 굵게 유지한다.

버튼 역할 가이드:
- Focus Start: 그린 또는 블루 포인트
- Give Up / Stop: 뉴트럴 그레이
- Collect Reward: 골드 포인트
- Retry: 뉴트럴 그레이 또는 소프트 코랄

## Iconography And Decoration
- 뼈다귀, 반짝이, 색종이 조각 같은 가벼운 장식 요소는 허용한다.
- 장식은 `Success` 상태에서 가장 적극적으로 사용한다.
- 평상시 화면에서는 장식보다 여백과 캐릭터에 집중한다.

## Motion
- 기본 상태 전환은 빠르지 않게, 부드럽게 처리한다.
- Idle -> Focusing
  - 캐릭터 교체와 배경색 전환 정도의 가벼운 애니메이션
- Focusing -> Success
  - 스케일 업, confetti, 보상 카드 등장 애니메이션 허용
- Focusing -> Fail
  - 짧은 페이드 다운 또는 살짝 아래로 처지는 모션

모션 원칙:
- 과한 bounce 남용 금지
- 타이머 동작 자체보다 상태 전환 순간을 강조
- 200ms~400ms 범위의 짧고 부드러운 전환 우선

## Flutter UI Guidance
- 초기 구현에서는 실제 일러스트가 부족해도 상태별 `색상 패널 + placeholder 이미지` 구조를 유지한다.
- 메인 화면은 하나의 큰 카드 안에 모든 핵심 정보를 넣는다.
- 추후 에셋 교체를 고려해 상태별 일러스트 영역을 독립 컴포넌트로 분리한다.

권장 컴포넌트 분리:
- `PetStatusHeader`
- `PetIllustrationCard`
- `FocusTimerDisplay`
- `SessionActionButton`
- `RewardSummaryCard`

## Do
- 크림톤 배경과 화이트 카드 중심의 부드러운 분위기를 유지한다.
- 상태별 색 역할을 명확히 나눈다.
- 타이머 숫자를 가장 먼저 보이게 만든다.
- 성공 보상 순간에만 시각적 밀도를 높인다.
- 캐릭터의 감정 표현을 UI 상태 변화보다 우선한다.

## Don't
- 네온, 고채도, 강한 대비 중심의 생산성 앱 톤으로 가지 않는다.
- 상시 장식 요소를 많이 넣어 화면을 복잡하게 만들지 않는다.
- 실패 상태를 공격적이거나 죄책감 유도형으로 표현하지 않는다.
- 성공 외 상태에서 금색 효과를 남용하지 않는다.
- 복잡한 카드 중첩과 많은 버튼으로 게임 UI처럼 만들지 않는다.

## Notes
- 구현 시작 전 `sample1.png`를 기준으로 색상 토큰과 상태별 컴포넌트 이름을 코드에 맞춰 정리하면 이후 에셋 교체가 쉬워진다.
- 실제 말티즈 에셋이 들어오면 이 문서의 색 역할, 레이아웃 우선순위, 상태별 감정 표현 원칙은 유지하고 세부 스타일만 조정한다.
